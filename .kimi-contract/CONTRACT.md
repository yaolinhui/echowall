## 📋 项目概述

**EchoWall** 是一个轻量级社交证明工具，用于自动抓取全网好评并生成可嵌入网站的口碑墙。

- **技术栈**：NestJS + React + TypeScript
- **后端**：`backend/` - NestJS API 服务
- **前端**：`frontend/` - React 管理后台
- **组件**：`widget/` - 可嵌入的口碑墙组件

## 📝 Git 提交规范

### 提交信息格式

```text
当前时间：中文描述

- 详细变更点 1
- 详细变更点 2

涉及文件：
- 文件路径 1
- 文件路径 2
```

### 示例

```bash
git commit -m "2026-03-18 16:36：修复 Microsoft Edge Tools 可访问性警告

- ProjectDetail.tsx：为 3 个 select 元素添加 title 属性
- demo.html：将 36 处内联样式重构为 CSS 类

涉及文件：
- frontend/src/pages/ProjectDetail.tsx
- widget/demo.html"
```

### 规则

1. **时间格式**：`yyyy-MM-dd HH:mm`
2. **语言**：必须使用中文
3. **首行**：简要概括本次提交的主要目的
4. **空行后**：列出具体的变更点，使用 `-` 符号
5. **涉及文件**：最后列出修改的文件路径

## 💻 代码风格约定

### TypeScript / React

1. **使用函数组件**，避免类组件
2. **类型定义**：优先使用 `type` 而非 `interface`
3. **导入顺序**：

   - React 相关
   - 第三方库
   - 项目内部模块
   - 类型定义

4. **命名规范**：

   - 组件：PascalCase（如 `ProjectDetail.tsx`）
   - 函数/变量：camelCase
   - 常量：UPPER_SNAKE_CASE

### CSS / Tailwind

1. **优先使用 Tailwind 类名**
2. **避免内联样式** `style={{}}`
3. **自定义样式**放到外部 CSS 文件

### 可访问性 (A11y)

1. **表单元素**必须有 `label` 或 `title` 属性
2. **图片**必须有 `alt` 属性
3. **交互元素**必须支持键盘操作

## 🗣️ 沟通约定

### 用户指令理解

1. **主动执行**：用户说"提交github"时，应自动执行 git add → commit → push 完整流程
2. **不需要确认**：对于约定好的规范操作（如提交），不需要额外询问
3. **需要确认**：对于以下情况必须询问用户：

   - 删除文件或数据
   - 修改配置文件（如 .env、docker-compose.yml）
   - 安装新的依赖包
   - 修改他人编写的核心逻辑

### 回复风格

1. **简洁明了**：直接给出结果，避免冗长解释
2. **使用表格**：多项目对比时使用表格展示
3. **代码块**：技术内容使用代码块格式化
4. **Emoji 使用**：适度使用，提升可读性

## 🔧 工作流程

### 开发流程

1. **理解需求** → 2. **修改代码** → 3. **验证结果** → 4. **提交 GitHub**

### 文件操作

1. **读取前**：优先使用 Glob 搜索，避免盲目读取
2. **修改后**：验证修改结果（如 Grep 确认）
3. **批量替换**：使用 StrReplaceFile 的多 edit 功能

### 错误处理

1. **遇到错误**：先尝试自行解决（如路径问题、依赖缺失）
2. **无法解决**：向用户报告具体错误信息
3. **修复后**：记录问题和解决方案到本文件「问题记录」部分

## 🎓 Markdown 编写规范（经验教训）

### 错误案例与正确写法

#### 1. 代码块必须指定语言 (MD040)

❌ **错误：**

```markdown
```
当前时间：中文描述
```
```

✅ **正确：**

```markdown
```text
当前时间：中文描述
```
```

#### 2. 标题前后必须有空行 (MD022)

❌ **错误：**

```markdown
### 规则
1. 第一条
```

✅ **正确：**

```markdown
### 规则

1. 第一条
```

#### 3. 列表前后必须有空行 (MD032)

❌ **错误：**

```markdown
导入顺序：
- 第一项
- 第二项
```

✅ **正确：**

```markdown
导入顺序：

- 第一项
- 第二项
```

#### 4. 强调标记内部不能有空格 (MD037)

❌ **错误：**

```markdown
** emoji 使用**
```

✅ **正确：**

```markdown
**Emoji 使用**
```
