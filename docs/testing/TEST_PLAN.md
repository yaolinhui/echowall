## 一、测试策略总览

```
┌─────────────────────────────────────────────────────────────┐
│                     测试金字塔                                │
├─────────────────────────────────────────────────────────────┤
│  冒烟测试 (Smoke)   │   Sprint开始，快速验证核心流程         │
├─────────────────────────────────────────────────────────────┤
│  单元测试 (Unit)    │   开发阶段，目标覆盖率>80%              │
├─────────────────────────────────────────────────────────────┤
│  集成测试 (Integration) │ API/模块间交互测试                  │
├─────────────────────────────────────────────────────────────┤
│  E2E测试 (End-to-End) │ 完整用户旅程测试                     │
├─────────────────────────────────────────────────────────────┤
│  性能测试 (Performance) │ 负载/压力/峰值测试                   │
├─────────────────────────────────────────────────────────────┤
│  安全测试 (Security) │ 渗透测试/漏洞扫描                     │
└─────────────────────────────────────────────────────────────┘
```

### 1.1 测试优先级定义

| 优先级 | 定义 | 响应时间 | 示例 |
|--------|------|----------|------|
| **P0 - 阻塞** | 核心功能，阻塞发布 | 必须修复 | 用户无法登录、Widget不显示 |
| **P1 - 高** | 主要功能，影响体验 | Sprint内修复 | 提及审核失败、数据源配置错误 |
| **P2 - 中** | 次要功能，有替代方案 | 下个版本修复 | 统计图表显示异常 |
| **P3 - 低** | 优化项 | 排期修复 | 界面样式微调 |

### 1.2 测试类型矩阵

| 测试类型 | 优先级 | 自动化 | 执行频率 | 负责人 |
|---------|--------|--------|----------|--------|
| 冒烟测试 | P0 | 半自动 | 每次提交 | 开发+测试 |
| 单元测试 | P0 | 全自动 | 每次提交 | 开发 |
| 接口测试 | P1 | 全自动 | 每日构建 | 测试/开发 |
| UI功能测试 | P1 | 半自动 | 提测后 | 测试 |
| 集成测试 | P1 | 半自动 | 提测后 | 测试 |
| 性能测试 | P2 | 全自动 | 上线前/每周 | 测试 |
| 安全测试 | P2 | 工具扫描 | 上线前 | 安全/测试 |
| 回归测试 | P1 | 全自动 | 每次发布前 | 自动化 |

Feature: 用户完整使用流程

Feature: 多平台数据整合

  Scenario: 合并GitHub和Product Hunt数据
    Given 项目已添加GitHub源
    And 项目已添加ProductHunt源
    When 同时触发两个源抓取

    And 不同平台mention正确标识

### 3.2 异常流程测试

| 场景ID | 异常场景 | 触发条件 | 系统行为 | 验证点 |
|--------|----------|----------|----------|--------|
