# 近似重复检测算法对比分析

## 概述

本文档详细对比各种近似重复检测算法的原理、优缺点和适用场景，帮助选择适合的去重策略。

---

## 1. SimHash (局部敏感哈希)

### 原理

1. 将文档分词，计算每个词的权重
2. 每个词通过哈希函数生成固定位数的二进制哈希
3. 对每一位，如果为1则加上权重，为0则减去权重
4. 最终正数为1，负数为0，得到SimHash
5. 相似文档的汉明距离小

### 数学表达

```
h(D) = Σ(wi × hi)
其中：
- wi 是第i个词的权重
- hi 是第i个词的哈希值
- 最终结果：sign(h(D)) → {0,1}^n
```

### 汉明距离与相似度

| 汉明距离 | 相似度估计 | 说明 |
|---------|-----------|------|
| 0 | 100% | 完全相同 |
| 1-3 | 95-99% | 高度相似 |
| 4-6 | 90-95% | 较相似 |
| 7-10 | 85-90% | 可能相似 |
| >10 | <85% | 不相似 |

### 优点
- 计算速度快，O(n)
- 支持LSH快速索引
- 内存占用小
- 适合大规模Web去重

### 缺点
- 只能检测较小差异（汉明距离有限）
- 对编辑不鲁棒（开头修改影响大）
- 对短文本效果差

### 适用场景
- 搜索引擎网页去重
- 大规模文档去重
- 实时去重系统

## 2. MinHash (最小哈希)

### 原理

1. 将文档转换为shingle集合
2. 使用k个哈希函数，对每个shingle计算哈希
3. 对每个哈希函数，取最小值作为签名
4. 两个文档的Jaccard相似度 ≈ 签名匹配比例

### 数学表达

```
J(A,B) = |A ∩ B| / |A ∪ B|

MinHash签名：
h_min(D) = [min(h1(D)), min(h2(D)), ..., min(hk(D))]

相似度估计：
sim(A,B) ≈ |{i | h_min(A)[i] = h_min(B)[i]}| / k
```

### LSH加速

```
将签名分成b个bands，每个band有r行
两个文档如果在一个band上完全匹配，则是候选对

捕获概率：P = 1 - (1 - s^r)^b
其中s是Jaccard相似度
```

### 优点
- 直接估计Jaccard相似度
- 对集合操作鲁棒
- LSH支持高效查找
- 理论保证充分

### 缺点
- 签名较长（通常128-256位）
- 构建成本较高
- 对文档长度敏感

### 适用场景
- 文档集合相似度
- 抄袭检测
- 重复代码检测

## 3. BERT/Sentence-BERT Embeddings

### 原理

1. 使用预训练语言模型编码文本
2. 生成固定维度的语义向量
3. 通过余弦相似度计算语义相似度

### 数学表达

```
Embedding: f(text) → R^d

余弦相似度：
cos(u,v) = (u·v) / (||u|| × ||v||)

语义相似度阈值通常：0.85-0.95
```

### 模型选择

| 模型 | 维度 | 速度 | 准确性 | 适用场景 |
|-----|-----|------|-------|---------|
| all-MiniLM-L6-v2 | 384 | 快 | 中 | 通用、实时 |
| all-mpnet-base-v2 | 768 | 中 | 高 | 高质量要求 |
| multi-qa-MiniLM | 384 | 快 | 中 | 问答系统 |
| paraphrase-multilingual | 384 | 中 | 高 | 多语言 |

### 优点
- 捕获语义相似性
- 对改写鲁棒
- 支持跨语言
- 准确性最高

### 缺点
- 计算成本高
- 需要GPU加速
- 向量维度高
- 需要向量数据库

### 适用场景
- 语义去重
- 问答系统
- 内容推荐
- 跨语言匹配

## 4. 算法综合对比

| 维度 | SimHash | MinHash | BERT |
|-----|---------|---------|------|
| **时间复杂度** | O(n) | O(n×k) | O(n²) |
| **空间复杂度** | O(1) | O(k) | O(d) |
| **准确性** | 中 | 高 | 最高 |
| **速度** | 最快 | 快 | 慢 |
| **语义理解** | 无 | 无 | 强 |
| **鲁棒性** | 中 | 高 | 最高 |
| **可扩展性** | 优秀 | 良好 | 中等 |

## 5. 混合策略建议

### 大规模生产环境 (>1000万条)

```
L0: Bloom Filter → 快速排除
L1: 精确Hash → 100%匹配
L2: SimHash → 候选筛选
L3: MinHash → 精确相似度
L4: BERT → 最终验证
```

### 中小规模 (<100万条)

```
L1: 精确Hash
L2: BERT Embeddings
```

### 资源受限环境

```
SimHash + LSH 单层方案
```

shingleSize = text.length < 100 ? 2 : 4;

numHashes = dataSize > 1M ? 256 : 128;

## 8. 常见问题

### Q: SimHash和MinHash如何选择？

A: 
- 需要语义理解 → BERT
- 大规模、实时 → SimHash
- 高精度要求 → MinHash
- 综合场景 → 混合使用

### Q: 如何处理多语言内容？

A:
- 使用多语言BERT模型
- 统一预处理（Unicode归一化）
- 语言检测后分别处理

### Q: 如何降低误报率？

A:
- 多层验证
- 调整阈值
- 增加人工审核环节
- 使用对抗样本训练
