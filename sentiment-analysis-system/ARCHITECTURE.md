# 多语言情感分析系统架构文档

## 系统概述

本系统是一个生产级的多语言情感分析解决方案，采用分层架构设计，在准确率、成本和延迟之间取得平衡。

## 核心组件

### 1. 路由决策器 (Router)

智能路由算法根据以下因素决策：

| 因素 | 权重 | 说明 |
|------|------|------|
| 文本复杂度 | 0.35 | 基于词汇、结构、情感复杂度 |
| 语言类型 | 0.20 | 中文/英文/混合语言 |
| 讽刺指标 | 0.25 | 是否存在讽刺线索 |
| 用户配置 | 0.20 | 强制指定层级或选项 |

**路由规则：**
- 复杂度 < 0.3 → 规则引擎
- 0.3 ≤ 复杂度 < 0.6 → 本地快速模型
- 0.6 ≤ 复杂度 < 0.8 → 本地标准模型
- 复杂度 ≥ 0.8 或 需要讽刺检测 → 云端LLM

### 2. 规则引擎 (RuleEngine)

**词典体系：**
- 中文情感词典: 500+ 词条
- 英文情感词典: 800+ 词条
- 技术领域词典: 200+ 专业术语
- 网络用语词典: 100+ 流行语

**处理逻辑：**
1. 分词和预处理
2. 否定词检测 (not, 不, 没)
3. 强度词加权 (very, 很, 非常)
4. 情感极性计算
5. 结果归一化

### 3. 本地模型 (LocalModel)

**支持的模型：**

| 模型 | 参数量 | 语言 | 特点 |
|------|--------|------|------|
| XLM-RoBERTa-base | 270M | 多语言 | 平衡方案 |
| XLM-RoBERTa-large | 550M | 多语言 | 高精度 |
| bert-base-chinese | 102M | 中文 | 快速 |
| distilbert-multilingual | 66M | 多语言 | 轻量 |

**优化手段：**
- ONNX 转换加速推理
- 动态批处理 (Dynamic Batching)
- 模型量化 (INT8)
- 缓存推理结果

### 4. 云端大模型 (CloudLLM)

**支持的提供商：**

| 提供商 | 模型 | 输入价格 | 输出价格 |
|--------|------|----------|----------|
| OpenAI | GPT-4o | $2.5/MTok | $10/MTok |
| OpenAI | GPT-4o-mini | $0.15/MTok | $0.6/MTok |
| Anthropic | Claude 3.5 Sonnet | $3/MTok | $15/MTok |
| Google | Gemini 1.5 Pro | $3.5/MTok | $10.5/MTok |

**提示工程：**
- System Prompt: 定义专家角色和分析标准
- Few-shot: 提供示例增强理解
- JSON Mode: 结构化输出便于解析

### 5. 讽刺检测器 (SarcasmDetector)

**多层检测：**

1. **规则层**: 检测讽刺词汇、标点模式、表情符号
2. **嵌入层**: 语义不一致性检测 (正面词+负面语境)
3. **上下文层**: 对话历史分析 (负面→正面转换)
4. **LLM层**: 深度推理确认

**检测指标：**
- 词汇线索权重: 0.25
- 标点模式权重: 0.15
- 情感冲突权重: 0.30
- 上下文权重: 0.15
- LLM确认权重: 0.15

## 安全考虑

1. **输入过滤**: 防止提示注入攻击
2. **速率限制**: 防止滥用和 DDoS
3. **数据脱敏**: PII 检测和处理
4. **访问控制**: API Key 和 JWT 认证
